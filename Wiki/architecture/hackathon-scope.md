# Hackathon Scope

## Themes
- Task-aware routing across models, tools, and policies.
- Agentic workflows across identity, security, infrastructure, and execution.

## Project Constraints
- Must be demoable under time pressure.
- Must degrade gracefully when parts fail.
- Should minimize agent token waste during implementation and demo prep.
- Should prefer local, explicit context over broad ambient prompting.

## Non-Goals
- Building a giant autonomous agent framework.
- Exhaustive long-form documentation.
- Dependence on fragile live-only integrations for core value.

## Success Criteria
- A new agent can orient quickly from preload pages.
- The team can answer architecture and implementation questions without broad repo spelunking.
- The demo clearly shows routing and agentic orchestration, not just generic chat behavior.
