# Legacy compatibility shim.
#
# Do not use this file for fresh installs.
# The training pipeline is intentionally split into two environments:
#
#   pip install -r requirements-train.txt
#   pip install -r requirements-serve.txt
#
# Why:
# - Unsloth / TRL / PEFT are the training stack.
# - vLLM / FastAPI are the serving stack.
# - Combining both in one env causes pip backtracking and binary conflicts.
#
# This file points at the training env only so accidental installs do not
# drag vLLM into the training environment.

-r requirements-train.txt
